1. 이미지 이름
컬럼의 길이를 조정할 필요가 있다. 퍼포먼스를 신경 쓰려면, UK 필요한지? 확인 해봐야 한다.

2. 보드 테이블
보트의 타입별 테이블을 만들어 주는게 좋다.
예를 들어 상품테이블, 그냥 보드 테이블, 이런식으로 나누어 지는게 좋다

3. DB 컬럼에 기본값
기본값이 없는 컬럼을 찾아서 작성해 준다

4. 업데이트 컬럼 약어
requested_at 은 at 가 있는데 updated는 없다 추가 작업 해야한다.

5. char를 사용 했으면 짧게
char를 사용 하여 컬럼값을 주었을때 간단하게 y 혹은 n 으로 표기 되게끔 하는게 좋다.
이것또한 퍼포먼스를 생각 해야한다.

6. 결재 관련 information 컬럼
기존 JSON 에서 괜히 바꾼듯 하다 그대로 사용해도 될 듯 하다.

7. int unsignd?
음수가 못들어오게 하는 컬럼 값인데
사용 하려면 front에서 많은 확인이 필요하다 되도록 사용 하지 말아야 한다.

8. DB 를 중고 거래가 쇼핑몰로 생각을 해야한다.
이전에 생각했던 쇼핑몰 플로우를 반영한다.

9. 마일리지 테이블 다시 생각

10. 예외처리에 var_dump
var_dump로 작성된것 수정하기

11. 헤드에 리콰이어
헤드 영역에 레이아웃 불러온것들 실제 레이아웃이 필요한지 보고 필요 없으면 오토 로드만 가져오면 된다.

12. 각 예외처리 이름
예외처리한 이름들 통일성 부족 rollback을 할거면 DatabaseException을 이용해서 다른 페이지도 똑같이 적용한다.

13. 세션 $auth 부분
